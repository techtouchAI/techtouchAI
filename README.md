# تطبيقات للشاشات (App Store CMS)

هذا المشروع عبارة عن منصة مفتوحة المصدر (بدون قواعد بيانات تقليدية) لعرض وتحميل التطبيقات والملفات الكبيرة (مثل APK, ZIP, RAR, EXE). يعتمد المشروع كلياً على **واجهة برمجة تطبيقات جيتهاب (GitHub API)** لتخزين البيانات والملفات، مما يجعله مجانياً وسهل الاستضافة.

## ✨ مميزات المشروع

* **بدون قاعدة بيانات:** يتم حفظ جميع البيانات (التطبيقات والإعدادات) في ملفات JSON داخل مستودع GitHub الخاص بك.
* **رفع ملفات غير محدود:** يستفيد المشروع من ميزة **GitHub Releases** لرفع الملفات الكبيرة (مثل التطبيقات، الألعاب، أو البرامج) متجاوزاً قيود الحجم المعتادة للملفات في المستودعات (مما يحل مشكلة رفع ملفات APK الكبيرة).
* **لوحة تحكم (CMS) متكاملة:** واجهة سهلة الاستخدام لإضافة، تعديل، وحذف التطبيقات، بالإضافة إلى تخصيص مظهر الموقع (مثل تغيير الاسم والشعار).
* **دعم الوضع الليلي:** تصميم عصري ومتجاوب يدعم الوضع المظلم والفاتح تلقائياً أو يدوياً.
* **بحث فوري:** ميزة البحث السريع عن التطبيقات في الصفحة الرئيسية.
* **تخزين مؤقت وتحديثات سريعة:** استخدام أحدث تقنيات Next.js لتوفير تجربة مستخدم سريعة جداً.

## 🛠️ التقنيات المستخدمة

* **إطار العمل:** [Next.js 15](https://nextjs.org/) (React 19)
* **التصميم:** [Tailwind CSS 4](https://tailwindcss.com/)
* **الأيقونات:** [Lucide React](https://lucide.dev/)
* **قاعدة البيانات والتخزين:** [GitHub REST API (Octokit)](https://github.com/octokit/octokit.js)

## 🚀 كيفية التثبيت والتشغيل محلياً

**المتطلبات الأساسية:** يجب أن يكون لديك [Node.js](https://nodejs.org/) مثبتاً على جهازك.

1. **نسخ المشروع وتثبيت الحزم:**
   ```bash
   git clone <رابط_المستودع>
   cd ai-studio-applet
   npm install
   ```

2. **تشغيل المشروع:**
   ```bash
   npm run dev
   ```
   سيتم تشغيل الموقع محلياً على الرابط: `http://localhost:3000`

## ⚙️ طريقة إعداد وربط GitHub (للوحة التحكم)

لكي يعمل الموقع وتتمكن من نشر التطبيقات، يجب ربطه بمستودع GitHub الخاص بك.

1. قم بإنشاء مستودع (Repository) جديد في حسابك على GitHub (يفضل أن يكون Public إذا كنت تريد أن يرى الناس الملفات، أو Private وتأكد من صلاحيات الوصول). **هام جداً: يجب ألا يكون المستودع فارغاً تماماً. قم بإنشاء ملف `README.md` واحد على الأقل فيه عند إنشائه.**
2. اذهب إلى إعدادات حسابك في GitHub: `Settings` > `Developer settings` > `Personal access tokens` > `Tokens (classic)`.
3. اضغط على **Generate new token (classic)**.
4. أعطِ الرمز اسماً (مثلاً: App Store CMS Token).
5. **الصلاحيات المطلوبة:** حدد مربع **`repo`** بالكامل (Full control of private repositories). هذه الصلاحية ضرورية لإنشاء الملفات، رفع الصور، وإنشاء الإصدارات (Releases) للملفات الكبيرة.
6. اضغط Generate وانسخ الرمز السري (Token).
7. افتح لوحة التحكم في الموقع من خلال الرابط: `http://localhost:3000/cms`
8. في تبويب **"إعدادات الاتصال"**، أدخل:
   * **GitHub Username:** اسم المستخدم الخاص بك (مثال: myusername).
   * **Repository Name:** اسم المستودع الذي أنشأته (مثال: my-app-store).
   * **Personal Access Token:** الرمز الذي نسخته في الخطوة 6.
9. اضغط على **حفظ واتصال**.

## 📝 كيفية نشر تطبيق جديد (طريقة عمل الرفع)

بعد نجاح الاتصال، سيظهر لك تبويب **"إدارة التطبيقات"**.

1. **الاسم والوصف:** أدخل اسم التطبيق ووصفه.
2. **صورة التطبيق:** اختر صورة للتطبيق (يُفضل ألا تتجاوز 5MB لسرعة الرفع). سيتم تحويل الصورة إلى نص (Base64) ورفعها إلى المستودع.
3. **ملفات التطبيق (السر الحقيقي 💡):**
   * عند اختيارك لملف (أو عدة ملفات مثل APK، EXE، ZIP)، **لا يتم** رفعها كملفات عادية داخل المستودع (لأن GitHub يحد حجم الملفات العادية بـ 100MB ويجعل المستودع بطيئاً).
   * بدلاً من ذلك، يقوم الكود (في خلفية لوحة التحكم) بإنشاء **Release (إصدار)** جديد مخفي في المستودع يحمل اسم التطبيق.
   * ثم يتم رفع الملفات الكبيرة كـ **Release Assets** داخل هذا الإصدار.
   * هذه الطريقة تسمح برفع ملفات كبيرة جداً (تصل إلى 2GB للملف الواحد) بدون أي مشاكل أو أخطاء وبسرعة تحميل ممتازة للزوار.
4. **النشر:** اضغط على **رفع ونشر**. سيظهر شريط تقدم يوضح حالة رفع الملفات الكبيرة. بمجرد الانتهاء، سيتم تحديث ملف `apps.json` في مستودعك وسيظهر التطبيق فوراً في الصفحة الرئيسية.

## 🎨 تخصيص الموقع

من خلال لوحة التحكم `/cms` > تبويب **"المظهر والإعدادات"**، يمكنك:
* تغيير اسم الموقع الذي يظهر في الأعلى.
* رفع شعار مخصص للموقع (يتم تخزينه أيضاً في GitHub).
